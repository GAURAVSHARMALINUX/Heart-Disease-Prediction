# Containerization and Kubernetes Deployment

## 1. Model Serving via FastAPI
The inference logic is encapsulated in a robust FastAPI web server (`api/app.py`). 
- **Endpoint:** POST `/predict`
- **Input:** JSON payload conforming to a Pydantic schema (representing patient health data).
- **Output:** Returns the prediction (`0` or `1`) along with the confidence score (probability).

## 2. Docker Containerization
The API is packaged into a Docker container for environment consistency.
- **Base Image:** `python:3.9-slim` (lightweight and secure).
- **Build Process:** Copies requirements, installs dependencies, runs data download and model training during the build phase (or expects pre-built models in a production setting), and exposes port 8000.

## 3. Kubernetes Deployment & Helm
The `k8s/deployment.yaml` manifest defines the deployment structure for scaling the application locally (via Minikube or Docker Desktop) or on cloud platforms.
- **Raw Manifests:** Spins up 2 replicas of the API container to ensure high availability and exposes it via a LoadBalancer.
- **Helm Chart:** A production-grade Helm chart is located in `helm/heart-disease-api/`. This allows for templated configurations, dynamic scaling, and simplified rollbacks using `helm install`.

## 4. Monitoring
The FastAPI application is fully instrumented using `prometheus-fastapi-instrumentator`.
- **Metrics Endpoint:** Real-time request and performance metrics are automatically exposed at the `/metrics` path.
- This allows seamless integration with Prometheus and Grafana for comprehensive system monitoring and alerting.
